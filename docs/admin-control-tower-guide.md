# 관리자 페이지(Admin) 보완 가이드 — 보안 · 컨트롤 타워 · UI

현재 구조를 기준으로, **보안 보완**, **꼭 있어야 할 기능**, **플로우/UI 정리** 방향을 정리했습니다.

---

## 1. 보안 보완

### 1.1 현재 상태
- **인증**: `requireAdminUser()`로 세션/역할 확인 후 레이아웃·API 접근 제한
- **관리자 지정**: `ADMIN_EMAILS` + 이메일 allowlist로 DB `User.role = ADMIN` 부여
- **API**: `/api/admin/*` 라우트마다 `requireAdminUser()` 호출

### 1.2 보완 권장 사항

| 항목 | 현재 | 권장 |
|------|------|------|
| **관리자 전용 경로** | `/admin`만 역할로 차단 | IP allowlist(선택), 관리자 전용 도메인/서브경로(예: `admin.kstay.com`) 검토 |
| **민감 API 이중 확인** | 없음 | 정산 출금·대량 거절·유저 역할 변경 등에 재인증(비밀번호/OTP) 또는 확인 모달 |
| **감사 로그(Audit Log)** | 없음 | “누가, 언제, 무엇을” 남기는 테이블 추가 후, 승인/거절/취소/정산 등 중요 행동 기록 |
| **세션/토큰** | NextAuth 기본 | 관리자 세션 타임아웃 짧게, 필요 시 관리자 전용 세션 정책 |
| **Rate limit** | 미적용 가능성 | `/api/admin/*` 전역 또는 중요 엔드포인트별 rate limit (봇/탈취 계정 남용 방지) |

## 2. 꼭 있어야 하는 기능 (컨트롤 타워)

### 2.1 이미 있는 것
- 요약(호스트 수, 승인 숙소, 승인 대기)
- 숙소 관리(승인/거절)
- 예약 관리(목록·상태 필터)
- 정산(결제완료 + 체크인 24h 기준)
- 고객센터(문의)
- 게시판/호스트 공지/KSTAY Black/테스트 리뷰

### 2.2 추가하면 좋은 핵심 기능

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **취소 이력 시각화** | 예약 취소 시 **누가(게스트/호스트)** **언제** 취소했는지 목록·상세에서 한눈에 표시 | 높음 |
| **관리자용 대시보드 요약 확장** | 오늘/이번 주 예약 수, 취소 수, 신규 호스트, 미처리 문의 수, 정산 대기 금액 등 카드/차트 | 높음 |
| **예약 상세(관리자 뷰)** | 예약 한 건 클릭 시 게스트/호스트/숙소/결제/취소 사유·시각 등 전체 정보 + 필요 시 메모 | 높음 |
| **Audit Log(활동 로그)** | “관리자 A가 숙소 B 승인”, “호스트 C가 예약 D 취소” 등 이벤트 목록 + 필터(날짜, 액터, 타입) | 중간 |
| **호스트/게스트 단일 뷰** | 한 유저 클릭 시 해당 유저의 숙소·예약·문의·리뷰 요약 | 중간 |
| **정산 출금 실행 로그** | “언제 어떤 예약에 대해 출금 요청했는지” 기록(포트원 연동 후) | 중간 |
| **알림/배지** | 상단 네비에 “승인 대기 3건”, “미답변 문의 5건” 등 배지로 표시 | 낮음 |

## 3. 플로우·UI 정리

### 3.1 정보 계층
- **1단계: 요약(대시보드)**  
  - 숫자 카드: 호스트 수, 승인 숙소, 승인 대기, **최근 취소 건수**, **미처리 문의** 등  
  - “빠른 작업”은 지금처럼 유지하되, **최근 취소** 링크 추가 권장  
- **2단계: 카테고리별 목록**  
  - 숙소 / 예약 / 정산 / 고객센터 등 — 현재처럼 탭/네비로 구분 유지  
- **3단계: 상세/액션**  
  - 목록에서 한 건 선택 → 상세 패널 또는 상세 페이지 → 승인/거절/메모 등

### 3.2 네비게이션 그룹 제안
- **운영**  
  - 요약, 숙소 관리, 예약 관리, 정산  
- **콘텐츠·정책**  
  - KSTAY Black, 게시판 관리, 호스트 공지  
- **고객 대응**  
  - 고객센터  
- **테스트/도구**  
  - 테스트 리뷰  

그룹별로 구분선이나 헤딩만 넣어도 “컨트롤 타워” 느낌이 나고, “예약/취소”는 운영 쪽에 묶이게 됩니다.

### 3.3 예약 관리 화면 보강
- **필터**: 상태(전체/결제대기/결제완료/취소) 유지 + **취소 주체**(전체/게스트 취소/호스트 취소) 필터 추가  
- **테이블 컬럼**:  
  - **상태** 옆에 “취소 주체”(게스트 측 취소 / 호스트 측 취소 / —)  
  - **취소 시각**(또는 “최종 변경 시각”) 컬럼 추가  
- **상세 진입**: 예약 ID 또는 행 클릭 → 관리자용 예약 상세(게스트/호스트/숙소/결제/취소 정보 일괄 표시)

## 4. 예시: “호스트가 취소하면 관리자 페이지에 시각적으로” — 구현 방향

### 4.1 데이터
- 이미 **Booking**에 `cancelledBy`(`GUEST` | `HOST`), `updatedAt` 있음  
- 취소 시각은 **updatedAt**으로 근사 가능. 더 정확히 하려면 **cancelledAt** 컬럼 추가 권장.

### 4.2 관리자 예약 목록
- **API**: `GET /api/admin/bookings` 응답에  
  - `cancelledBy`, `updatedAt`(또는 `cancelledAt`) 포함  
- **목록 UI**  
  - 상태가 “취소”인 행에  
    - 배지: “게스트 측 취소” / “호스트 측 취소”  
    - 툴팁 또는 부가 컬럼: “2025-03-02 14:30 취소” 형태  
- **필터**  
  - “취소” 선택 시, 서브 필터로 “전체 / 게스트 취소 / 호스트 취소” 제공

### 4.3 요약(대시보드) 카드
- **“최근 취소”** 카드 추가  
  - 예: 최근 7일간 취소 건수  
  - 클릭 시 `/admin/bookings?status=CANCELLED` 또는 “최근 취소” 필터로 이동  
- 선택: “오늘 취소 N건” 등 기간별 요약

### 4.4 (선택) 취소 전용 뷰
- **“취소 이력”** 메뉴 또는 “예약 관리” 내 탭  
  - 기간·취소 주체(게스트/호스트) 필터  
  - 테이블: 예약 ID, 숙소, 게스트, 호스트, 체크인/아웃, **취소 주체**, **취소 시각**  
  - 분쟁 대응 시 “누가 언제 취소했는지” 한 화면에서 확인 가능

---

## 5. 구현 우선순위 제안

1. **단기**  
   - 관리자 예약 API/목록에 `cancelledBy` + 취소 시각(updatedAt 또는 cancelledAt) 반영  
   - 예약 목록 테이블에 “취소 주체” 컬럼 + “게스트 측 취소”/“호스트 측 취소” 배지  
   - 필요 시 Booking에 `cancelledAt` 추가 후 취소 처리 시점에 기록  

2. **중기**  
   - 관리자 대시보드에 “최근 취소” 카드 및 예약 관리 링크  
   - 예약 한 건 상세 페이지(관리자 뷰)  
   - 네비 메뉴 그룹(운영/콘텐츠/고객대응/테스트) 정리  

3. **장기**  
   - Audit Log 테이블 + 중요 액션 로깅  
   - 정산 출금 실행 로그  
   - Rate limit / 관리자 세션 정책  

이 순서로 적용하면 “호스트가 취소하면 관리자 페이지에 시각적으로 누가 언제 취소했는지 보이게” 하는 부분은 빠르게 달성하고, 그 위에 보안과 컨트롤 타워 기능을 단계적으로 쌓을 수 있습니다.
