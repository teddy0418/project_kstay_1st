# 메시지함 & 고객센터 티켓 테스트 가이드

## 사전 준비

1. **앱 실행**
   ```bash
   npm run dev
   ```
   브라우저에서 `http://localhost:3001` (또는 설정한 포트) 접속.

2. **로그인**
   - **실제 로그인**: Google 등으로 로그인 후 테스트.
   - **데모 모드**: `.env.local`에 `KSTAY_ENABLE_DEMO_AUTH=true` 추가 후, 쿠키로 역할 전환 가능.  
     (역할 전환 UI가 있다면 게스트/호스트/관리자로 바꿔가며 테스트.)

---

## 1. 고객센터 티켓 (가장 간단)

**게스트/호스트 → 고객센터 문의**

1. 로그인한 상태에서 **도움말** 이동: 하단/메뉴에서 도움말 또는 `/help` 접속.
2. **「문의」** 섹션의 **「고객센터 문의하기」** 클릭 → `/support` 이동.
3. **「새 문의하기」** 클릭 → 제목 + 내용 입력 후 **보내기**.
4. 문의 목록에 티켓이 보이고, 클릭하면 상세(메시지 스레드) 확인 가능.
5. 같은 계정으로 메시지 추가 발송 가능(USER 역할).

**관리자 → 답변 시 문의자 알림**

1. **관리자**로 로그인: `/admin` 접속 가능한 계정(또는 데모 시 역할을 ADMIN으로).
2. 상단 **「고객센터」** 클릭 → `/admin/support`.
3. 목록에서 방금 만든 티켓 선택 → `/admin/support/[ticketId]`.
4. 하단 입력창에 답변 입력 후 **보내기** (SUPPORT 역할로 전송).
5. **문의한 계정(게스트/호스트)**으로 다시 로그인 → 알림에 고객센터 답변 알림 표시되는지 확인.  
   (알림 목록/벨 아이콘 등에서 `supportTicketId` 기반으로 해당 티켓으로 이동 가능해야 함.)

## 2. 예약 메시지 (호스트 ↔ 게스트)

**필요 조건**: 상태가 **CONFIRMED**인 예약이 최소 1건 있어야 함.

- 시드/스크립트로 예약이 있다면 그대로 사용.
- 없다면: 게스트로 예약 플로우 진행 후 결제(테스트 결제)까지 완료해 CONFIRMED 예약 생성.

**게스트**

1. **게스트** 계정으로 로그인.
2. **메시지** 이동: 하단 네비 **「메시지」** 또는 `/messages`.
3. 로그인 안 되어 있으면 「로그인해 주세요」 문구 표시.
4. CONFIRMED 예약이 있으면 **예약 건별 스레드 목록** 표시 (숙소명, 체크인/아웃, 마지막 메시지 요약).
5. 한 스레드 클릭 → `/messages/booking/[bookingId]` → 메시지 목록 + 입력창에서 **보내기** (GUEST 메시지).

**호스트**

1. **호스트** 계정으로 로그인 (해당 숙소의 호스트여야 함).
2. **호스트 대시** → **메세지** 또는 `/host/messages`.
3. 내 숙소의 CONFIRMED 예약 스레드 목록 표시.
4. 스레드 클릭 → `/host/messages/booking/[bookingId]` → 게스트 메시지 확인 후 **답변 입력 → 보내기** (HOST 메시지).
5. **게스트** 계정으로 다시 로그인 → 해당 예약 스레드에 호스트 답변 표시 + **알림**에 호스트 답변 알림 표시되는지 확인.

---

## 3. 알림 확인

- 알림 API: `GET /api/notifications` → `unreadCount`, `notifications[]`.
- 타입: `HOST_REPLIED`(예약 메시지), `SUPPORT_REPLIED`(고객센터 답변).
- 알림 클릭 시:
  - `bookingId` 있으면 → `/messages/booking/[bookingId]`
  - `supportTicketId` 있으면 → `/support/[ticketId]`

(알림 목록/벨 UI가 있다면 위 경로로 이동하도록 링크 연결해 두면 테스트하기 좋음.)

---

## 4. 한 번에 흐름으로 요약

| 순서 | 누가 | 어디서 | 할 일 |
|------|------|--------|--------|
| 1 | 게스트/호스트 | `/help` → 고객센터 문의하기 | 새 문의 작성 (제목+내용) |
| 2 | 관리자 | `/admin/support` | 해당 티켓 열어서 답변 작성 |
| 3 | 게스트/호스트 | 알림 또는 `/support` | 고객센터 답변 도착 확인 |
| 4 | 게스트 | `/messages` | CONFIRMED 예약 스레드에서 메시지 전송 |
| 5 | 호스트 | `/host/messages` | 해당 스레드에서 답변 전송 |
| 6 | 게스트 | `/messages` 또는 알림 | 호스트 답변 및 알림 확인 |

---

## 5. API로만 빠르게 확인

- **고객센터 티켓 목록**: `GET /api/support/tickets` (로그인 필수)
- **티켓 생성**: `POST /api/support/tickets` body: `{ "subject": "테스트", "body": "내용" }`
- **예약 스레드 목록**: `GET /api/messages/bookings` (로그인 필수)
- **알림**: `GET /api/notifications` (로그인 필수)

(Fetch/Postman 등에서 쿠키에 세션 쿠키 포함해 요청.)
