# KSTAY — 고민·보완·로직 정리

정산 주기 검토와 함께, 프로젝트 전반에서 결정이 필요한 부분·보완 포인트·로직상 필요한 것들을 정리한 문서입니다.

---

## 1. 정산 관련 (고민 필요)

| 항목 | 현재 | 결정/고민 포인트 |
|------|------|------------------|
| **정산 가능 시점** | 체크인 + 24시간 경과 (하드코딩) | 24시간을 그대로 둘지, 설정 가능하게 할지 (env/config). |
| **정산 주기** | 없음 (UI에만 "화요일 정산" 문구) | 주간 정산 시 **요일**(화요일 등)·**시간**(예: 02:00 KST)·**배치 실행** 방식 결정 필요. |
| **정산 단위** | 예약 단위로 "정산 가능"만 계산 | 주기마다 **한 번에 묶어서** 출금할지, 예약별로 개별 출금할지 정책 결정. |
| **출금 실행** | 버튼만 있고 API 미연동 | PortOne(또는 PG) 출금/송금 API 연동 시점·테스트 환경 정리. |
| **정산 내역 저장** | 없음 (Booking + Payment만 존재) | "이번 주에 이 금액을 호스트에게 지급했다"는 **정산 이력 테이블**(Settlement 또는 Payout) 필요 여부. 있으면 재정산·감사·세금 대응에 유리. |

---

## 2. 보완하면 좋은 부분 (MVP/플레이스홀더)

| 영역 | 현재 | 보완 시 기대 |
|------|------|--------------|
| **실제 OAuth** | 구글 로그인 버튼 → 데모 세션 | Google Client ID/Secret 설정 후 실제 계정 연동. (docs/GOOGLE_LOGIN.md 참고) |
| **이메일 로그인** | UI만, DB 저장 없음 | 이메일 인증·비밀번호 또는 매직링크 연동. |
| **검색/필터** | 전체 목록 또는 ids | 키워드·지역·날짜·가격 필터 API + 브라우즈/검색 UI 연결. |
| **호스트 정산 화면** | "정산 내역/출금 요청 (MVP)" 문구만 | 정산 주기 확정 후, 정산 예정액·지급 완료 내역·출금 요청 플로우. |
| **호스트 알림** | "(MVP) 정산/예약/운영 알림" 문구만 | 실시간/목록 알림 (예약 신청, 취소, 정산 완료 등). |
| **호스트 계정** | 정산 계좌 UI 있음, 프로필은 LocalStorage 안내 | 세금 정보(사업자번호 등) 입력 필드 필요 시 추가. |
| **관리자 정산** | 목록·정산 가능 여부만 표시 | 실제 정산 배치 실행·엑셀 다운로드·주간 지급 리스트 생성 (현재 버튼은 "준비중"). |
| **리뷰** | DB 저장·조회·작성 폼 있음 | 예약 완료 게스트만 리뷰 가능 제한·노출 일관성 점검. |
| **메시지** | 페이지만 존재 | 게스트–호스트 메시지 함, 읽음/알림. |
| **지도** | MVP 안내 문구 | Google Maps API 키 연동 후 지도 팝업 교체. |
| **프로필(게스트)** | 이름·소개 등 LocalStorage | 구글 로그인 연동 후 계정별 서버 저장 전환. |

## 3. 로직적으로 필요한 부분

| 항목 | 현재 | 필요한 것 |
|------|------|------------|
| **게스트 예약 취소** | 취소 정책·무료 취소 기한은 UI/정책에 있음. **게스트가 "예약 취소"를 누르는 플로우·API는 없음** | 게스트가 무료 취소 기한 내에서 취소 요청 → 예약 상태 CANCELLED + (결제 취소/환불 처리). 무료 기한 이후 취소 시 환불 불가 안내 또는 부분 환불 정책 적용. |
| **취소 시 결제 환불** | 호스트 거절 시 CANCELLED로만 변경. PG 취소/환불 API 호출 여부는 PortOne 웹훅·수동 취소 플로우 확인 필요 | 게스트 취소·호스트 거절 시 **실제 PG 취소/환불** 호출이 있어야 함. (MOCK 모드는 즉시 확정이므로 별도 처리 가능) |
| **정산 이력** | Booking·Payment만으로 "정산 가능" 계산. "언제 얼마를 지급했는지" 기록 없음 | 정산 실행 시 **Settlement(또는 Payout) 테이블**에 회차·호스트·금액·상태·지급일 저장하면 재정산·문의·감사 대응 가능. |
| **승인 API 상태 검증** | 관리자 승인 API가 listing id만 받고 **PENDING 여부를 검사하지 않음** | `status === "PENDING"` 일 때만 approve/reject 허용하도록 API에서 검증 추가 권장. |
| **정산 가능 24시간** | repository에서 `readyAt = checkIn + 24h` 고정 | 운영 중 변경 가능성이 있으면 **설정값(env 또는 config)** 으로 빼두는 것이 좋음. |

## 4. 우선순위 제안 (참고)

- **지금 결정/정리해 두면 좋은 것**  
  - 정산 주기(요일·시간)·정산 단위(주간 묶음 vs 예약별).  
  - 게스트 예약 취소 플로우 유무 및 무료 취소 기한 내만 허용할지 여부.

- **구현 우선 제안**  
  - 게스트 예약 취소 API + (선택) 프로필/예약 내역에서 취소 버튼 연결.  
  - 관리자 승인 API에 `status === "PENDING"` 검증 추가.  
  - 정산 주기 확정 후: 정산 이력 테이블 설계, 배치/출금 연동.

- **사업자등록·PG 이후**  
  - 실제 결제·취소/환불·정산 출금 테스트 및 PortOne 연동 정리.

## 5. 한 줄 요약

- **고민**: 정산 주기(요일/시간), 정산 단위(묶음 vs 개별), 정산 가능 24시간 설정화, 정산 이력 저장 여부.  
- **보완**: OAuth/이메일 로그인, 검색/필터, 호스트 정산·알림·계정 실기능, 메시지, 지도, 프로필 서버 저장.  
- **로직**: 게스트 예약 취소 플로우 + 취소 시 환불 처리, 정산 이력 저장, 관리자 승인 시 PENDING 검증.
